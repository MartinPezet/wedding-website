## Why

The physical stationery should share the website's floral identity: RSVP letters carrying each party's personal QR code, a full-size printed seating chart, place cards, and on-the-day handouts — all generated by the app and printed via the browser.

## What Changes

- Shared print layer: print CSS + floral design tokens reused from the site, page-size presets, crop marks where needed
- RSVP letters: A5, one per party, personal QR code (party token URL), party names, invite copy; batch view prints every party sequentially via print-to-PDF
- Seating chart print: the persisted seating layout rendered at large format (A2/A1) in site style with table names and guest names
- Place cards: A4 grids with fold lines and crop marks — guest name + meal marker per card
- Day handouts: A5 order-of-the-day (and similar) rendered from `handout.json` in site style

## Capabilities

### New Capabilities

- `print-materials`: browser-printable stationery — shared print layer, QR invite letters, large-format seating chart, place cards, and JSON-driven handouts

### Modified Capabilities

_None._

## Impact

- Depends on `add-rsvp-system` (parties/tokens/meals), `add-seating-chart` (layout data), `add-admin-panel` (admin auth for print routes)
- New dependency: `qrcode` (SVG QR generation)
- New content file: `handout.json`
